This project implements a RAG chatbot that answers questions about Hong Kong labour regulations using reference documents. The Documents are parsed, split into overlapping text chunks, embedded using Google's gemini-embedding-001 model, and stored in a persistent Chroma vector database. At query time, the system uses Maximal Marginal Relevance retrieval to retrieve relevant document chunks, which are then passed to gemini-3.5-flash to generate responses, along with source document and page number citations. While the system performs well on direct and regulation-specific questions, retrieval quality can degrade for broad, multi-document, or highly nuanced queries because the current implementation relies solely on dense vector search without reranking or hybrid keyword retrieval. Future improvements could include hybrid retrieval (BM25 + vector search), cross-encoder reranking, and metadata-aware filtering to improve recall and citation accuracy
